# ERP Module Checklists

Use this reference when planning or reviewing a specific ERP module. Each checklist is a starting point, not a complete spec. Add company-specific policies, regulations, and integration details.

## 1. Sales And Order Management

### Core Process

1. 고객/거래처 등록
2. 영업 기회 또는 견적 등록
3. 견적 승인
4. 계약 또는 주문 생성
5. 출고/납품 요청
6. 매출 확정
7. 세금계산서/청구
8. 수금 및 미수 관리

### Planning Checklist

- 고객과 거래처를 구분해야 하는가?
- 견적 유효기간과 버전 관리가 필요한가?
- 할인율, 마진, 특가 승인 기준이 있는가?
- 주문 확정 후 수정/취소 정책은 무엇인가?
- 여신 한도, 미수금, 거래 중지 상태를 체크해야 하는가?
- 출고 가능 여부를 재고와 연동해야 하는가?
- 세금계산서 발행 기준일은 주문일, 출고일, 검수일, 매출 확정일 중 무엇인가?
- 영업 담당자 변경 시 과거 실적 귀속은 어떻게 처리하는가?

### Key Reports

- 영업 파이프라인
- 견적 전환율
- 수주/매출 현황
- 미수금 현황
- 담당자별 실적

## 2. Purchase And Procurement

### Core Process

1. 구매 요청
2. 예산/필요성 검토
3. 견적 수집 및 비교
4. 구매 승인
5. 발주
6. 입고/검수
7. 매입 확정
8. 지급 요청

### Planning Checklist

- 구매 요청과 발주를 분리해야 하는가?
- 품목 구매와 비용 구매를 구분해야 하는가?
- 금액별 승인 단계가 다른가?
- 예산 초과 시 차단, 경고, 추가 승인 중 무엇인가?
- 공급사 선정 이력과 견적 비교표가 필요한가?
- 입고 전 선지급, 부분 입고, 초과 입고를 허용하는가?
- 3-way match가 필요한가? 발주, 입고, 세금계산서 기준.
- 긴급 구매 예외 프로세스가 있는가?

### Key Reports

- 구매 요청/발주 현황
- 공급사별 구매 금액
- 미입고/지연 입고
- 예산 대비 구매
- 단가 변동 이력

## 3. Inventory And Logistics

### Core Process

1. 품목/창고 기준정보 관리
2. 입고
3. 출고
4. 창고 이동
5. 재고 조정
6. 실사
7. 반품/폐기
8. 재고 마감

### Planning Checklist

- 품목 단위, 보조 단위, 포장 단위가 필요한가?
- LOT, Serial, 유통기한 관리가 필요한가?
- 음수 재고를 허용하는가?
- 창고별, 위치별, 상태별 재고를 관리하는가?
- 가용재고 계산에서 예약, 보류, 불량, 검수대기를 제외하는가?
- 재고 조정은 승인 후 반영되는가?
- 실사 차이는 자동 조정인가, 승인 후 조정인가?
- 재고 원가와 회계 반영이 필요한가?

### Key Reports

- 품목별 재고 현황
- 창고별 재고 현황
- 입출고 이력
- 장기재고/부진재고
- 실사 차이

## 4. Production And MES Link

### Core Process

1. BOM/공정 기준정보 관리
2. 생산계획 수립
3. 작업지시 생성
4. 자재 출고
5. 공정 실적 입력
6. 불량/재작업 처리
7. 생산 완료 입고
8. 원가/실적 집계

### Planning Checklist

- BOM 버전 관리와 적용일이 필요한가?
- 작업지시는 수동 생성인가, 수주/계획 기반 자동 생성인가?
- 자재 투입은 표준 소요량 기준인가, 실제 투입 기준인가?
- 공정별 실적, 설비, 작업자, 시간을 기록하는가?
- 불량 유형과 귀책을 관리하는가?
- 재작업과 폐기 프로세스가 분리되는가?
- MES와 ERP 중 어느 시스템이 기준 데이터인가?
- 생산 완료가 재고와 회계 원가에 어떤 시점에 반영되는가?

### Key Reports

- 생산계획 대비 실적
- 공정별 생산성
- 불량률
- 자재 소요/부족
- 표준 원가 대비 실제 원가

## 5. Accounting, Settlement, And Closing

### Core Process

1. 거래 발생
2. 증빙 수집
3. 전표 생성
4. 검토/승인
5. 지급/수금
6. 정산
7. 마감
8. 보정/역분개

### Planning Checklist

- 자동 전표와 수동 전표를 구분해야 하는가?
- 비용센터, 프로젝트, 법인, 계정과목 기준이 필요한가?
- 증빙 첨부와 증빙 검증 규칙이 있는가?
- 전표 승인과 지급 승인은 분리되는가?
- 마감 이후 수정은 금지, 마감 해제, 보정 전표 중 무엇인가?
- 세금계산서, 카드, 은행, PG 연동이 필요한가?
- 지급 계좌 변경과 지급 실행 권한은 분리되어 있는가?
- 정산 결과와 원천 거래의 대사가 필요한가?

### Key Reports

- 전표 현황
- 비용 현황
- 미지급/미수
- 정산 결과
- 마감 체크리스트

## 6. HR, Attendance, And Payroll

### Core Process

1. 직원 기준정보 관리
2. 조직/직책/직무 관리
3. 근태 입력/수집
4. 휴가/초과근무 신청
5. 승인
6. 급여 계산
7. 급여 확정
8. 지급 및 신고

### Planning Checklist

- 조직 변경 이력을 기준일별로 관리해야 하는가?
- 근무제, 교대, 탄력근무, 재택근무 정책이 있는가?
- 휴가 발생, 사용, 차감, 이월 규칙은 무엇인가?
- 근태 수정은 누가 승인하는가?
- 급여 항목별 계산식과 예외가 필요한가?
- 급여 데이터 접근 권한은 어떻게 제한하는가?
- 퇴사자, 휴직자, 겸직자의 처리는 어떻게 다른가?
- 소급 적용이 필요한가?

### Key Reports

- 인원 현황
- 근태 현황
- 휴가 잔여
- 초과근무
- 급여 지급 현황

## 7. Approval And Workflow

### Core Process

1. 문서 작성
2. 결재선 지정
3. 상신
4. 검토/합의
5. 승인/반려
6. 후속 업무 실행
7. 보관/검색

### Planning Checklist

- 결재선은 고정, 조직도, 금액, 문서 유형, 수동 지정 중 무엇인가?
- 합의, 참조, 수신, 시행 기능이 필요한가?
- 대결, 위임, 전결 정책이 있는가?
- 회수와 재상신 조건은 무엇인가?
- 반려 시 이전 단계로 돌아가는가, 기안자에게 돌아가는가?
- 승인 완료 후 문서 수정은 가능한가?
- 결재 완료가 다른 ERP 업무를 자동 실행하는가?
- 첨부파일 접근 권한은 결재 문서 권한과 동일한가?

### Key Reports

- 결재 대기/지연
- 문서 유형별 처리량
- 평균 승인 시간
- 반려율
- 사용자별 처리 현황

## 8. CS And Operations

### Core Process

1. 문의 접수
2. 분류
3. 담당자 배정
4. 처리
5. 에스컬레이션
6. 답변/조치
7. 종료
8. 재문의/품질 평가

### Planning Checklist

- 문의 채널은 전화, 이메일, 채팅, 웹폼, 쇼핑몰 중 무엇인가?
- 고객/계약/주문 정보와 연동되는가?
- 자동 분류 또는 AI 요약이 필요한가?
- SLA 기준과 위반 알림이 있는가?
- 보상, 환불, 교환은 별도 승인되는가?
- 담당자 변경과 에스컬레이션 이력을 남기는가?
- 민감 문의 접근 권한을 제한해야 하는가?
- 종료 후 재오픈이 가능한가?

### Key Reports

- 문의 접수/처리량
- 평균 처리 시간
- SLA 준수율
- 유형별 이슈
- 재문의율

## 9. Reporting And BI

### Core Process

1. 지표 정의
2. 데이터 소스 확인
3. 집계 기준 설계
4. 권한별 조회 범위 정의
5. 대시보드/리포트 구성
6. 다운로드/구독/알림
7. 정합성 검증

### Planning Checklist

- KPI 정의와 계산식이 명확한가?
- 기준일, 집계 기간, 조직 기준, 법인 기준이 정해졌는가?
- 원천 데이터와 리포트 데이터의 차이를 설명할 수 있는가?
- 실시간, 준실시간, 일배치 중 무엇이 필요한가?
- 사용자가 필터, 피벗, 드릴다운을 해야 하는가?
- 리포트 다운로드 권한과 마스킹 정책이 필요한가?
- 리포트 수치의 검증 책임자는 누구인가?
- 경영진용 요약과 실무자용 상세가 분리되는가?

### Key Reports

- 경영 대시보드
- 부서별 KPI
- 예외/이상탐지 리포트
- 월마감 리포트
- 데이터 정합성 리포트
