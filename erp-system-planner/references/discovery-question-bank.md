# Discovery Question Bank

Use this reference to generate sharp questions for ERP discovery, system analysis, and improvement planning. Select only questions relevant to the task.

## 1. Universal Questions

- 이 업무의 시작 조건은 무엇인가요?
- 이 업무가 완료되었다고 판단하는 기준은 무엇인가요?
- 현재 이 업무는 어떤 시스템, 엑셀, 메신저, 이메일, 종이 문서로 처리하나요?
- 가장 시간이 많이 걸리는 단계는 어디인가요?
- 가장 자주 오류가 나는 단계는 어디인가요?
- 누가 최종 책임자인가요?
- 승인, 검토, 참조, 처리 역할이 분리되어 있나요?
- 마감 이후 수정이나 취소는 가능한가요?
- 예외 상황은 누가 어떤 기준으로 판단하나요?
- 이 업무 결과가 영향을 주는 다음 업무는 무엇인가요?

## 2. Questions For Business Users

- 실제로 하루/주/월 몇 건 정도 처리하나요?
- 시스템 밖에서 따로 관리하는 엑셀이나 메모가 있나요?
- 저장 전에 알고 있는 정보와 나중에 알게 되는 정보는 무엇인가요?
- 화면에서 가장 찾기 어려운 정보는 무엇인가요?
- 반복 입력하거나 복사/붙여넣기하는 항목은 무엇인가요?
- 오류가 생겼을 때 어떻게 복구하나요?
- 승인 대기나 처리 지연을 어떻게 확인하나요?
- 업무가 몰리는 시점은 언제인가요?
- 현재 리포트에서 신뢰하지 않는 수치가 있나요?
- 개선된다면 가장 먼저 줄이고 싶은 작업은 무엇인가요?

## 3. Questions For Managers And Approvers

- 승인 기준은 금액, 부서, 품목, 거래처, 위험도 중 무엇에 따라 달라지나요?
- 어떤 건은 자동 승인해도 되고, 어떤 건은 반드시 검토해야 하나요?
- 승인자가 확인해야 하는 핵심 정보는 무엇인가요?
- 반려 사유는 정형화할 수 있나요?
- 승인 지연 시 대체 승인자나 에스컬레이션이 필요한가요?
- 마감 이후 수정 요청은 누가 승인해야 하나요?
- 대시보드에서 매일 봐야 하는 지표는 무엇인가요?
- 부서별 또는 담당자별 성과를 어떻게 보고 있나요?

## 4. Questions For Operators/Admins

- 코드값, 양식, 결재선, 권한은 누가 관리하나요?
- 사용자 권한 변경 요청은 어떤 절차로 처리하나요?
- 관리자 강제 수정이 필요한 경우가 있나요?
- 시스템 장애나 연동 실패는 어떻게 감지하나요?
- 실패한 배치나 API는 누가 재처리하나요?
- 엑셀 업로드 오류는 어떻게 안내하나요?
- 공지, 알림, 템플릿 문구는 누가 수정하나요?
- 운영자가 가장 자주 처리하는 문의는 무엇인가요?

## 5. Questions For Developers

- 현재 데이터 모델에서 기준정보와 거래 데이터가 분리되어 있나요?
- 상태값은 코드화되어 있나요, 텍스트로 관리되나요?
- 삭제는 물리 삭제인가요, 논리 삭제인가요?
- 변경 이력은 어떤 테이블/로그에 남나요?
- 외부 연동 실패 시 재처리 구조가 있나요?
- API는 멱등성을 보장하나요?
- 권한은 프론트엔드와 백엔드 모두에서 검증되나요?
- 대량 조회, 다운로드, 업로드 성능 이슈가 있나요?
- 테스트 데이터와 운영 데이터가 분리되어 있나요?
- 배포 후 설정값 변경으로 해결 가능한 영역은 무엇인가요?

## 6. Questions For Finance, Audit, And Control

- 회계/정산/급여/지급에 영향을 주는 데이터는 무엇인가요?
- 작성자와 승인자, 승인자와 지급자가 분리되어야 하나요?
- 마감 후 보정 방식은 원거래 수정인가요, 보정 거래 생성인가요?
- 감사 시 반드시 제출해야 하는 이력은 무엇인가요?
- 전표, 세금계산서, 지급 데이터와 대사가 필요한가요?
- 권한 점검은 얼마나 자주 하나요?
- 엑셀 다운로드 이력과 개인정보 접근 이력이 필요한가요?
- 내부통제상 금지해야 하는 권한 조합은 무엇인가요?

## 7. Questions For Security And Privacy

- 개인정보, 급여, 계좌, 계약금액, 원가 등 민감 정보가 포함되나요?
- 민감 정보는 마스킹되어야 하나요?
- 다운로드, 인쇄, 복사 제한이 필요한가요?
- SSO, MFA, IP 제한, VPN 제한이 필요한가요?
- 퇴사자와 조직 변경자의 권한은 어떻게 회수되나요?
- 외부 협력사 계정이 접근하나요?
- 파일 첨부는 바이러스 검사, 다운로드 제한, 만료 링크가 필요한가요?

## 8. Questions For Executives

- 이 시스템 개선의 1순위 목표는 비용 절감, 속도 개선, 통제 강화, 데이터 가시성, 고객 경험 중 무엇인가요?
- 성공 여부를 어떤 KPI로 판단할까요?
- 이번 단계에서 반드시 포함해야 하는 범위와 제외해도 되는 범위는 무엇인가요?
- 표준 ERP 프로세스에 맞추는 것과 기존 업무 방식을 유지하는 것 중 어느 쪽이 더 중요한가요?
- 현업 변화관리와 교육에 어느 정도 리소스를 투입할 수 있나요?

## 9. AI And Automation Questions

- AI가 요약/추천만 해야 하나요, 초안 생성이나 저장까지 해도 되나요?
- AI 결과를 누가 검토하고 승인하나요?
- AI가 사용해도 되는 데이터와 금지되는 데이터는 무엇인가요?
- AI가 실행하면 안 되는 업무는 무엇인가요?
- AI 추천의 근거와 로그가 필요한가요?
- AI가 틀렸을 때 수정, 되돌리기, 책임 소재는 어떻게 처리하나요?
- 자동화 예외는 어떤 기준으로 사람에게 넘기나요?
